#### [SketchGPT вАФ QuickDraw Generation](https://github.com/SarahCho0/QuickDraw_Generation)
**Paper Implementation from Scratch вАФ Decoder-only Transformer for Sequential Sketch Generation**

`Individual` &nbsp;|&nbsp; `RTX 3090 ¬Ј AMP Training ¬Ј Google QuickDraw`

Full ground-up reimplementation of ¬І3.4 of the SketchGPT paper вАФ no reference codebase, built entirely from the paper spec.

**Architecture:** Decoder-only Transformer вАФ **8 layers ¬Ј 8 heads ¬Ј d_model=512** вАФ treating sketches as sequences of discrete stroke tokens rather than pixels. Pretrained on 10 vehicle classes from Google QuickDraw.

**The core challenge вАФ category bleed:** A single model pretrained across multiple classes conflated visual features (cars bleeding into planes, bikes into motorcycles). Solved by **per-class fine-tuning**: the shared backbone captures universal stroke grammar; class-specific heads learn category-distinct generation patterns.

**Full pipeline built from scratch:**
- **Stroke-3 format preprocessing** вАФ raw (x, y, pen_state) point sequences normalized and tokenized into primitive stroke units
- **AMP training** with `torch.cuda.amp` for mixed-precision on RTX 3090 вАФ ~2√Ч memory efficiency enabling larger batch sizes
- **Cosine LR scheduling** with warmup вАФ stabilizes transformer training on sequential stroke data
- **Class-conditional top-k sampling** at inference вАФ temperature-controlled diversity with per-class quality tuning

**Why this stands out:** Generative models for sequential vector data (vs. raster images) require fundamentally different tokenization, loss functions, and sampling strategies вАФ this project covers all of them end-to-end from a paper, not a tutorial.

`PyTorch` `CUDA` `Scikit-learn` `Google QuickDraw`
